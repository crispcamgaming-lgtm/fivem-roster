CREATE TABLE roster_members (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  rank TEXT,
  callsign TEXT,
  status TEXT DEFAULT 'active',
  discord_id TEXT,
  badge TEXT,
  join_date DATE,
  last_seen DATE,
  strikes INT DEFAULT 0,
  flags JSONB DEFAULT '[]',
  notes JSONB DEFAULT '[]',
  certs JSONB DEFAULT '[]',
  loa JSONB DEFAULT '[]',
  roster_id TEXT,
  access TEXT DEFAULT 'member',
  updated_at TIMESTAMPTZ DEFAULT now()
);

📄 GitHub Pages + Integration Setup Guide ▼ Show
① Host on GitHub Pages (Free)
  1. Create a GitHub repo (e.g. nexus-roster)
  2. Upload nexus-roster.html and rename it to index.html
  3. Go to repo Settings → Pages → Source: main branch / root
  4. Your site is live at username.github.io/nexus-roster
  5. All data saves in localStorage — no server required
② Supabase (Live Multi-User)
  1. Create free project at supabase.com
  2. SQL Editor → paste the schema from Integrations page
  3. Settings → API → copy Project URL and anon key
  4. Paste both into Supabase integration fields here
  5. Click Connect, then Push All to seed your data
  6. Enable Real-time subscription for live multi-user sync
③ Discord Application (OAuth2)
  1. Go to discord.com/developers/applications
  2. New Application → name it (e.g. Nexus Roster)
  3. OAuth2 → copy Client ID, generate Client Secret
  4. OAuth2 → Redirects → Add your GitHub Pages URL + /callback
  5. No bot token needed — this is OAuth2 login only
  6. Members see "Login with Discord" on the public roster page
④ Google Sheets Bidirectional
  1. Open your Google Sheet → Extensions → Apps Script
  2. Paste the Apps Script code (copy from below)
  3. Deploy → New deployment → Web app → Anyone access
  4. Copy the Web App URL into Sheets integration
  5. Add your Spreadsheet ID from the URL
  6. Push/Pull syncs your roster to that sheet instantly
